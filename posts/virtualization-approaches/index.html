history and use cases of virtualisation

A hypervisor is the solution that allows virtual machines to share host resources.

Virtualisation allows for running multiple operating systems on a single physical host.

This helps enhancing server utilisation and management.

Consilidation is sharing the resources of a (big) server by multiple virtual machines, leading to reduced cost

Paravartualisation

Paravirtualization is a technique to improve the performance, the guest operating system has support for paravirtualization to work more efficiently with the hypervisor with less overhead.

remember that the hypervisor modifies and redirects virtual machine’s calls to the hardware, for paravirtualization case calls are redirected without modification.

Another way to enhance performance is hardware assisted virtualization extension like Intel VT and AMD V, which offers additional instruction set to assist the hypervisor.

insights about hypervisors

hypervisors used by cloud providers:

  • AWS

    • EC2 instances use their own Nitro hypervisor (built on core KVM), but supports Xen-based VMs
    • Amazon Linux is available as virtual machine images for : KVM, Hyper-V, VirtualBox, and ESXi
    • It also supports running Hyper-V on a bare metal instance, or working with VMware machines

  • Azure uses a console only edition of windows (called Cloud Host) and Hyper-V, and have migraation options for VMware and Xen machines

  • GCP uses KVM with their own front end

  • Dedicated Host-HUAWEI CLOUD uses KVM

factors to decide which hypervisor to use vary from (license, price and, support) to (speed, ease of use, and security)

Before chosing hypervisor to rely on you need to specify your problem needs and priorities. For example you may want:

  • remotely centralized management and monitoring
  • moving or replicating virtual machines across servers
  • utilization of limited GPUs and resources on only some servers
  • Virtual networking
  • Storage management features
